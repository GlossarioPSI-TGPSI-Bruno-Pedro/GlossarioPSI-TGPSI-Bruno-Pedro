Instance:
  In object-oriented programming, it is called the instance of a class, an object
whose behavior and state are defined by the class.




Interface:
  In some programming languages, the term interface is a reference to
characteristics that allow the construction of interfaces that isolate the
implementation details of a software component.




Internal:
  It is an access specifier, it means this statement restricts access
which other classes have to class or variable that the internal constrains. There are several types of
access specifiers, internal restricts access to classes that derive from the class
where the internal, public specifies that a certain class or variable is of
free access to all other classes of the program, and private isolates the class or variable,
making it inaccessible.




Instance Methods:
  They are methods whose values can be changed during the execution of the
program.
